3. Explain Exploratory Data Analysis. (10)

Answer

EDA is the first step towards building a model. EDA is often relegated to chapter 1 (by which we mean the “easiest” and lowest level) of standard introductory statistics textbooks and then forgotten about for the rest of the book.

It’s traditionally presented as a bunch of histograms and stem-and-leaf plots. They teach that stuff to kids in fifth grade so it seems trivial, right? No wonder no one thinks much of it.

But EDA is a critical part of the data science process, and also represents a philosophy or way of doing statistics practiced by a strain of statisticians coming from the Bell Labs tradition.

John Tukey, a mathematician at Bell Labs, developed exploratory data analysis in contrast to confirmatory data analysis, which concerns itself with modeling and hypotheses as described in the previous section. In EDA, there is no hypothesis and there is no model. The “exploratory” aspect means that your understanding of the problem you are solving, or might solve, is changing as you go.

The basic tools of EDA are plots, graphs and summary statistics. Generally speaking, it’s a method of systematically going through the data, plotting distributions of all variables (using box plots), plotting time series of data, transforming variables, looking at all pairwise relationships between variables using scatterplot matrices, and generating summary statistics for all of them. At the very least that would mean computing their mean, minimum, maximum, the upper and lower quartiles, and identifying outliers.

But as much as EDA is a set of tools, it’s also a mindset. And that mindset is about your relationship with the data. You want to understand the data — gain intuition, understand the shape of it, and try to connect your understanding of the process that generated the data to the data itself. EDA happens between you and the data and isn’t about proving anything to anyone else yet.

Example

There are 31 datasets named nyt1.csv, nyt2.csv,…,nyt31.csv, which you can find here: https://github.com/oreillymedia/doing_data_science.

Each one represents one (simulated) day’s worth of ads shown and clicks recorded on the New York Times home page in May 2012. Each row represents a single user. There are five columns: age, gender (0=female, 1=male), number impressions, number clicks, and logged-in.

You’ll be using R to handle these data. It’s a programming language designed specifically for data analysis, and it’s pretty intuitive to start using. Once you have it installed, you can load a single file into R with this command:

data1 <- read.csv(url("http://stat.columbia.edu/~rachel/datasets/nyt1.csv"))

Once you have the data loaded, it’s time for some EDA:

  1. Create a new variable, age_group, that that categorizes users as “<18”, “18-24”, “25-34”, “35-44”, “45-54”, “55-64”, and “65+”.
  1. For a single day:
    1. Plot the distributions of number impressions and click-through-rate (Click Through Rate=Number of ClicksNumber of Impressions)\left( \text{Click Through Rate} = \frac{\text{Number of Clicks}}{\text{Number of Impressions}} \right)(Click Through Rate=Number of ImpressionsNumber of Clicks​)﻿ for these six age categories.
    1. Define a new variable to segment or categorize users based on their click behavior.
    1. Explore the data and make visual and quantitative comparisons across user segments/demographics (<18-year-old males versus <18-year-old females or logged-in versus not, for example).
    1. Create metrics/measurements/statistics that summarize the data. Examples of potential metrics include CTR, quantiles, mean, median, variance, and max, and these can be calculated across the various user segments. Be selective. Think about what will be important to track over time—what will compress the data, but still capture user behavior.
  1. Now extend your analysis across days. Visualize some metrics and distributions over time.
  1. Describe and interpret any patterns you find.

4. Which Machine Learning algorithm to be used when you have bunch of objects that are already classified and based on which other similar objects that haven’t got classified to be automatically labelled? Explain. (10)

Answer

Introduction:

k-Nearest Neighbors (k-NN) is a machine learning algorithm that is used when you have a set of objects that have already been classified or labeled, and you need to automatically label similar, yet unlabeled objects.

Description of k-NN:

  • k-NN is particularly useful for tasks where you need to classify new data points based on their similarity to previously labeled data.
  • For instance, objects could be classified into categories like "high credit" or "low credit," "high cancer risk" or "low cancer risk," etc.

Comparison with Linear Regression:

  • Linear regression is typically used for predicting continuous variables, not categorical labels.
  • However, with thresholds, linear regression can be adapted to create bins or ranges for classification, though this approach has limitations.
  • Unlike linear regression, k-NN directly handles categorical labels by voting among the nearest neighbors.

Intuition Behind k-NN:

  • The algorithm works by finding the most similar items (neighbors) based on their attributes.
  • The unlabeled object is then given the label that is most common among its nearest neighbors.
  • If there is a tie in votes, the label is selected randomly among those tied.

Practical Example with Credit Scores:

  • Scenario: You have data on individuals' age, income, and credit ratings (either "high" or "low").
  • Goal: Predict the credit rating for a new individual based on their age and income.

Steps in k-NN:

  1. Similarity Metric: Decide how to measure similarity or distance (e.g., Euclidean distance).
  1. Data Splitting: Divide the dataset into training and test sets.
  1. Evaluation Metric: Choose an evaluation metric like misclassification rate.
  1. Running k-NN: Test the algorithm with different values of k and choose the one that gives the best performance.
  1. Prediction: Once k is optimized, use it to predict labels for new data points.

6. Explain K-means algorithm with example. (6)

Answer

Introduction to k-Means:

  • k-Means is an unsupervised learning algorithm that aims to find natural groupings or clusters within data.
  • Unlike supervised learning, where the correct labels are known, in k-means, the algorithm itself defines the clusters based on the structure of the data.

Problem Statement:

  • Suppose you have a dataset where each row represents a user with attributes such as age, gender, income, state, and household size.
  • The goal is to segment these users into groups (clusters) that share similar characteristics, a process known as clustering, segmenting, or stratifying.

Applications of k-Means Clustering:

  • Personalization: Different users can be given personalized experiences, such as targeted marketing based on cluster membership (e.g., offering printer toner to those likely to own printers).
  • Model Improvement: Some models may perform better on specific groups, and different clusters might require different models or strategies.
  • Hierarchical Modeling: In statistics, k-means can help model different effects separately (e.g., geographical vs. household effects in survey data).

Manual Binning Example:

  • To understand the need for k-means, consider a manual approach to grouping users:
    • Binning Attributes: Users could be grouped into bins based on thresholds (e.g., age groups: 20–24, 25–30; income ranges; etc.).
    • Dimensionality Challenge: If you create bins for each attribute, you end up with a large number of possible combinations (e.g., 10 age bins × 2 gender bins × 50 state bins × 10 income bins × 3 household size bins = 30,000 possible bins).
    • High Dimensionality: The data can be thought of as existing in a multi-dimensional space, with each axis corresponding to an attribute (e.g., age, gender, income, etc.).
    • Need for Automation: Manually creating and managing such a large number of bins is impractical, especially in higher dimensions.

k-Means Clustering Approach:

  • Defining Clusters: k-means automates the clustering process by dividing the data into k clusters, where k is a predefined number of clusters.
  • Cluster Formation: The algorithm places data points into clusters by minimizing the distance between the data points and the centroid of the cluster they belong to.

Simplified 2D Example:

  • 2D Data Visualization: Consider a 2-dimensional dataset where each user has two features: the number of ads shown (impressions) and the number of clicks.
  • Visual Clustering: In a simple 2D space, clusters may be visually apparent, as shown in Figure 3-9. However, as the number of dimensions and data points increases, it becomes challenging to identify clusters without an algorithm.
  • High-Dimensional Clustering: k-means is effective in finding clusters in high-dimensional spaces (where d is the number of features).

7. Describe philosophy of EDA in detail. (8)

Answer

While at Google, Rachel was fortunate to work alongside two former Bell Labs/AT&T statisticians—Daryl Pregibon and Diane Lambert, who also work in this vein of applied statistics—and learned from them to make EDA a part of her best practices.

Yes, even with very large Google-scale data, they did EDA. In the context of data in an Internet/engineering company, EDA is done for some of the same reasons it’s done with smaller datasets, but there are additional reasons to do it with data that has been generated from logs.

There are important reasons anyone working with data should do EDA. Namely, to gain intuition about the data; to make comparisons between distributions; for sanity checking (making sure the data is on the scale you expect, in the format you thought it should be); to find out where data is missing or if there are outliers; and to summarize the data.

In the context of data generated from logs, EDA also helps with debugging the logging process. For example, “patterns” you find in the data could actually be something wrong in the logging process that needs to be fixed. If you never go to the trouble of debugging, you’ll continue to think your patterns are real. The engineers we’ve worked with are always grateful for help in this area.

In the end, EDA helps you make sure the product is performing as intended.

Although there’s lots of visualization involved in EDA, we distinguish between EDA and data visualization in that EDA is done toward the beginning of analysis, and data visualization, as it’s used in our vernacular, is done toward the end to communicate one’s findings. With EDA, the graphics are solely done for you to understand what’s going on.

With EDA, you can also use the understanding you get to inform and improve the development of algorithms. For example, suppose you are trying to develop a ranking algorithm that ranks content that you are showing to users. To do this you might want to develop a notion of “popular.”

Before you decide how to quantify popularity (which could be, for example, highest frequency of clicks, or the post with the most number of comments, or comments above some threshold, or some weighted average of many metrics), you need to understand how the data is behaving, and the best way to do that is looking at it and getting your hands dirty.

Plotting data and making comparisons can get you extremely far, and is far better to do than getting a dataset and immediately running a regression just because you know how. It’s been a disservice to analysts and data scientists that EDA has not been enforced as a critical part of the process of working with data. Take this opportunity to make it part of your process!

# Load the data
data1 <- read.csv(url("http://stat.columbia.edu/~rachel/datasets/nyt1.csv"))

# Categorize age into age groups
data1$agecat <- cut(data1$Age, c(-Inf, 0, 18, 24, 34, 44, 54, 64, Inf))

# View the summary of the data
summary(data1)

# Install and load the doBy package
install.packages("doBy")
library("doBy")

# Function to calculate summary statistics
siterange <- function(x) {
  c(length(x), min(x), mean(x), max(x))
}

# Summary of age by age category
summaryBy(Age ~ agecat, data = data1, FUN = siterange)

# Summary of various metrics by age category
summaryBy(Gender + Signed_In + Impressions + Clicks ~ agecat, data = data1)

# Install and load ggplot2 package
install.packages("ggplot2")
library(ggplot2)

# Plot histogram of Impressions by age category
ggplot(data1, aes(x = Impressions, fill = agecat)) +
  geom_histogram(binwidth = 1)

# Plot boxplot of Impressions by age category
ggplot(data1, aes(x = agecat, y = Impressions, fill = agecat)) +
  geom_boxplot()

# Create click-through rate variable
data1$hasimps <- cut(data1$Impressions, c(-Inf, 0, Inf))

# Summary of Clicks by hasimps
summaryBy(Clicks ~ hasimps, data = data1, FUN = siterange)

# Density plot of Clicks/Impressions by age category (for those with Impressions > 0)
ggplot(subset(data1, Impressions > 0), aes(x = Clicks / Impressions, colour = agecat)) +
  geom_density()

# Density plot of Clicks/Impressions by age category (for those with Clicks > 0)
ggplot(subset(data1, Clicks > 0), aes(x = Clicks / Impressions, colour = agecat)) +
  geom_density()

# Boxplot of Clicks by age category (for those with Clicks > 0)
ggplot(subset(data1, Clicks > 0), aes(x = agecat, y = Clicks, fill = agecat)) +
  geom_boxplot()

# Density plot of Clicks by age category (for those with Clicks > 0)
ggplot(subset(data1, Clicks > 0), aes(x = Clicks, colour = agecat)) +
  geom_density()

# Create categories based on Impressions and Clicks
data1$scode[data1$Impressions == 0] <- "NoImps"
data1$scode[data1$Impressions > 0] <- "Imps"
data1$scode[data1$Clicks > 0] <- "Clicks"

# Convert the scode column to a factor
data1$scode <- factor(data1$scode)

# View the first few rows of the data
head(data1)

clen <- function(x) {
  c(length(x))
